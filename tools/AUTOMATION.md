# Veil 회개 로컬응답 — 문제해결 자동화 (방법 기록)

회개 기능의 **로컬(무료) 응답 품질**을 반복 측정·수정으로 끌어올리는 자동화 루프.

## 엔진
- `tools/confession-eval.mjs` — 재현 가능한 평가 하니스.
  - `confession-data.js` + `confession-cases.js`를 읽어, **script.js의 매칭 로직을 그대로 미러링**한다.
  - 18개 카테고리 전체에 "사용자가 실제로 적을 법한" 자유서술 고백 배터리(38건)를 돌린다.
  - **PASS** = 구체 사례 매칭(CASE) + 카테고리가 허용집합 안 + (위기 입력이면) `care` 안내 존재.
  - 실행: `node tools/confession-eval.mjs` (전부 통과 시 exit 0, 실패 있으면 exit 1 + FAIL 목록).

## 루프 (3단계, 최대 15회)
1. **문제 파악** — 하니스 실행 → FAIL(GENERAL/오라우팅/care누락) 목록 확보.
2. **수정** — 우선순위: ① 데이터(카테고리·사례 키워드 보강, `confession-data.js`) ② 알고리즘(매칭 점수식·임계값).
3. **검증** — 하니스 재실행. 통과하든 안 하든 1번으로 복귀.
- **중단 조건**: 같은 문제에 3회 봉착 → 사람에게 보고 후 중단. 또는 15회 도달. 또는 100% 수렴.

## 동기화 규칙
- **데이터 수정**(confession-data.js / confession-cases.js)은 앱·하니스가 같은 파일을 읽으므로 자동 반영.
- **알고리즘 수정**은 `script.js`의 `matchCase`/`matchCategory`와 하니스를 **반드시 동시에** 고친다(현재 미러링 상태).

## 현재 매칭 알고리즘 (검증본)
- 글자 **bigram 겹침(overlap)** + 사례 키워드 보너스(0.07/회, 상한 0.20) + 카테고리 키워드 보너스(0.06/회, 상한 0.18).
- **2단계**: ① 카테고리 키워드로 유력 카테고리 판정(임계 0.20) ② 전체 최고 사례가 유력 카테고리와 다르면 그 카테고리 내 최고로 교정.
- 온디바이스·무비용·프라이버시 유지(고백이 기기 밖으로 나가지 않음).

## 이력 (2026-06-09)
| 회차 | 조치 | PASS |
|---|---|---|
| 0 | 데이터 구조·커버리지 파악(18카테고리·200사례, 무결성 OK) | — |
| 1 | 사례 점수에 카테고리 키워드 보너스 추가 | 71% → 79% |
| 2 | 7개 카테고리 판별 키워드 보강 | 79% → 89% |
| 3 | 2단계(카테고리 우선) 매칭 도입 | 89% → **100%** (배터리 38) |
| 4 | 배터리 38→66 확장(구어/오타/복합/위기) + 채점기준 정정(카테고리 정답 인정) + 튜닝(CAT_KW_W 0.12·CAT_CONF 0.16) + 7개 결함 키워드 보강 | 86% → 98% |
| 5 | 회귀 수정('험담'을 lie에서 제거, gossip 귀속) | 98% → **100%** (배터리 66) |
| 6 | 배터리 66→91 확장(초단문/장문/복합/위기) + 6개 결함 보강(위기 '죽음' care 라우팅 포함) | 93% → **100%** (배터리 91) |

현재 배터리 91건 전부 통과(구체사례 97%·위기 care 정상). 배터리를 늘릴수록 새 약점이 드러나므로, 회차를 거듭하며 배터리를 계속 확장하는 것이 정석. 단, 합성 배터리는 결국 내부 작성이라 **실사용자 로그가 가장 신뢰도 높은 다음 배터리원천**이다. 

## 향후 (권장)
- 배터리를 실사용자 표현으로 계속 확장(현재 38건은 내부 작성 → 과적합 주의).
- 필요 시 온디바이스 임베딩(Transformers.js)과 정확도 비교(Phase 3).

## 자동화2 — 라우팅 강건성 15라운드 (2026-06-10)
배터리를 ~93 → **245건**으로 확장(매 라운드 새 유형 ~10건 추가: 직장·돈·디지털·가정·성/연애·교회생활·영적침체·중독·감정·수치·**위기/안전**·초단문/슬랭·복합죄·모호토로·신학난이도·혼합). 각 라운드 실패를 그 자리에서 데이터 키워드로 수정 → 매 라운드 100% 복구.
- 발견·수정한 결함 클래스: '상처를 줬'(가해)↔불용서, '말씀'(시어머니 말씀)↔prayerless, '판단/평가'↔gossip, '의심'(자녀)↔doubt, '집착'↔greed, 'SNS/폰'↔sloth, '원망'(하나님)↔불용서, 그리고 **위기 표현('손목을 그어'·'끝내면 편해'·'극단적'·'폭력 견딜 수 없')의 care 누락** → 전부 despair(care 보유)로 라우팅.
- 하니스 버그도 수정: CAT 레벨 매칭 시 카테고리 care 미검증 → 수정(앱은 정상이었음).
- **최종: 245/245 = 100%, 구체사례 97%, GENERAL/오라우팅 0, 위기 care 정상.** 알고리즘 불변(전부 confession-data.js 키워드) → script.js 자동 동기화. 합성 배터리 한계는 여전 → 실사용자 로그가 다음 원천.
